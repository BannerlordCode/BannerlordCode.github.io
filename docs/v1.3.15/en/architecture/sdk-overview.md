# SDK Overview — Module Map & Developer Roadmap

## Mental Model

Treat `SDK Overview — Module Map & Developer Roadmap` as an entry point or data node for this subsystem: inspect its properties first, then decide which methods to call.

> This is the **navigation hub** of the whole documentation. It is not just a module list — it answers three questions from a developer's perspective: **why** each layer exists, **when** you will touch it, and **how** to jump from here to the relevant API docs. After reading this page you should be able to tell, in under a minute, "which layers does my task need, what's the entry class, and where is the doc".

## In one sentence

Bannerlord's managed code is a set of **strictly layered** `TaleWorlds.*` assemblies: foundation libraries → engine bridge → core data → UI framework → save system → game logic (Mission + Campaign) → platform/services. **90% of mod work happens in the top four layers — Campaign, Mission, UI, and Save; the bottom four are foundational and only touched directly for basics like reading input, doing math, or querying scenes.**

## Layered dependency diagram

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 8  Platform & Services                                 │
│   ModuleManager · Diamond · PlatformService(.Steam/.Epic/.GOG)│
│   PlayerServices · ServiceDiscovery · Launcher · Diamond(M&B) │
├─────────────────────────────────────────────────────────────┤
│ Layer 7  Campaign  ← top mod surface                          │
│   CampaignSystem · CampaignSystem.ViewModelCollection         │
├─────────────────────────────────────────────────────────────┤
│ Layer 6  MountAndBlade  ← top mod surface                     │
│   MountAndBlade · .AutoGenerated · .ViewModelCollection       │
│   .Helpers · .GauntletUI.Widgets                              │
├─────────────────────────────────────────────────────────────┤
│ Layer 5  SaveSystem  ← persistence                            │
│   SaveSystem (SaveManager / SaveContext / LoadContext)        │
├─────────────────────────────────────────────────────────────┤
│ Layer 4  Gauntlet UI                                          │
│   GauntletUI · .Data · .PrefabSystem · .ExtraWidgets · .CodeGen│
├─────────────────────────────────────────────────────────────┤
│ Layer 3  Engine bridge                                        │
│   Engine · .AutoGenerated · .GauntletUI · NavigationSystem    │
├─────────────────────────────────────────────────────────────┤
│ Layer 2  Core data                                            │
│   Core (ItemObject/CharacterObject/Equipment) · Core.VMCollection│
├─────────────────────────────────────────────────────────────┤
│ Layer 1  Systems base                                         │
│   ObjectSystem · ScreenSystem · InputSystem · DotNet ·        │
│   TwoDimension · Network                                      │
├─────────────────────────────────────────────────────────────┤
│ Layer 0  Foundation                                           │
│   Library · LinQuick · Localization                           │
└─────────────────────────────────────────────────────────────┘
        ↕ depends on Native engine (TaleWorlds.Native.dll) throughout
```

Dependencies flow **bottom-up**: upper layers reference lower ones, never the reverse. Mod code sits at the very top and may touch any layer. **New modders can start from the top four layers**; the bottom four are supporting infrastructure — look them up when a specific API sends you there.

## Developer roadmap

> This is the centerpiece of this doc. **"I want to do X — which layers do I touch?"** The table below maps common mod goals to layers, entry classes, and doc starting points. All links are relative so they work across versions/languages.

| I want to... | Touch these layers | Entry class / mechanism | Start here |
|--------------|--------------------|------------------------|-----------|
| Register a mod (game lifecycle hook) | L8 ModuleManager + L6 M&B | `MBSubModuleBase`, `ModuleInfo` | [Module System](./module-system) |
| Add a new troop / item / equipment | L2 Core + XML | `ItemObject`, `CharacterObject`, `Equipment` + `MBObjectManager` | [Core API](../api/core/) · [Items API](../api/items/) |
| Make a campaign mod (daily ticks, behaviors, kingdom logic) | L7 Campaign | `CampaignBehaviorBase` / `CampaignBehavior`, `CampaignEvents` | [Campaign API](../api/campaign/) · [Campaign Guide](../guide/campaign-system) |
| Make a combat / mission mod (AI, damage, spawning) | L6 MountAndBlade | `MissionBehavior`, `MissionLogic`, `Agent`, `Formation` | [Mission API](../api/mission/) · [Mission Guide](../guide/mission-system) |
| Add a custom UI screen (HUD, menu, dialog) | L4 Gauntlet + L6 M&B | `ViewModel` + `GauntletMovie`, `ScreenBase` | [GUI API](../api/gui/) · [Gauntlet Guide](../guide/gauntlet-ui) |
| Persist custom data across saves | L5 SaveSystem | `[SaveableField]` / `[SaveableProperty]` + `SaveManager` | [Save API](../api/save-system/) · [Save Architecture](./save-system) |
| Add localized text (multi-language strings) | L0 Localization | `TextObject`, `MBTextManager` | [Localization API](../api/localization/) · [Localization Guide](../guide/localization) |
| Manipulate scene entities (position, skeleton, effects) | L3 Engine | `Scene`, `GameEntity`, `MatrixFrame`, `Skeleton` | [Engine API](../api/engine/) |
| Read player input (keyboard / gamepad) | L1 InputSystem | `InputContext`, `InputKey` | [API Index](../api/) |
| Add a new ViewModel / data binding | L4 Gauntlet + L1 Core.VMCollection | `ViewModel`, `DataSourceProperty` | [ViewModel API](../api/viewmodel/) |
| Customize campaign extension types (settlements, templates) | L7 Campaign + L2 Core | `Settlement`, `PartyTemplateObject`, `CultureObject` | [Campaign-Ext API](../api/campaign-ext/) |
| Customize battle components / Agent behavior | L6 MountAndBlade extension | `AgentComponent`, `MissionLogic` subclasses | [Mission-Ext API](../api/mission-ext/) |
| Extend core models (crafting, skills, concepts) | L2 Core extension | `SkillObject`, `CraftingTemplate`, `BasicCharacterObject` | [Core-Extra API](../api/core-extra/) |
| Multiplayer底层 (rare; split in 1.4.5) | L8 Network + L6 M&B.Multiplayer | `MultiplayerMission` family | [Version Delta](./version-delta) |

> **How to use this table**: pick the row matching your goal → open the API doc and read the entry-class signature → come back here to check whether a dependent layer is needed (e.g. campaign mods almost always also need L5 Save).

## Module catalog (v1.3.15, grouped by layer)

> Scale: **54 TaleWorlds modules / 5196 .cs files / 5811 types**. ★ = new in 1.3.15. The "Mod relevance" column gives **concrete use cases** rather than abstract ratings, so you can decide whether to dig in.

### Layer 0 — Foundation

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.Library` | Utilities: `Vec3`/`Mat3`/`Vec2` math, `MBList`/`MBReadOnlyList`, threading, IO, callbacks, `InformationManager` messaging, debugging | Daily — almost every file `using TaleWorlds.Library`; info-message popups, vector math, debug logs |
| `TaleWorlds.LinQuick` | LINQ perf-optimized replacement | Internal, mods don't reference directly |
| `TaleWorlds.Localization` | Localization: `TextObject`, `MBTextManager`, language data | Every player-facing string flows through here; custom strings, i18n mods |

### Layer 1 — Systems base

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.ObjectSystem` | `MBObjectManager`, `MBObjectBase`, XML deserialization & object registry | Loading custom items/characters from XML; `MBObjectManager.RegisterObject` |
| `TaleWorlds.ScreenSystem` | Screen stack `ScreenBase`/`ScreenManager` | Custom full-screen UI, overriding default screens |
| `TaleWorlds.InputSystem` | Input abstraction `InputContext`, keys/gamepad | Reading player keys, custom hotkey mods |
| `TaleWorlds.DotNet` / `.AutoGenerated` | Managed/native interop primitives, readonly dicts | Internal, rarely used directly |
| `TaleWorlds.TwoDimension` / `.Standalone` | 2D rendering: sprites, fonts, canvas | Engine-internal; only for low-level 2D rendering mods |
| `TaleWorlds.Network` | Network primitives | Multiplayer mod base; more relevant after 1.4.5 multiplayer split |

### Layer 2 — Core data

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.Core` | Core domain: `ItemObject`/`Equipment`/`CharacterObject`/crafting/basic concepts/info channels | Defining new items, troops, equipment; touched by both campaign and mission mods |
| `TaleWorlds.Core.ViewModelCollection` | Core ViewModels | Used with L4 for core data binding |

### Layer 3 — Engine bridge

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.Engine` | Engine bridge: `Scene`, `GameEntity`, `NativeObject`, `MatrixFrame`, `Skeleton`, `Utilities` | Manipulating battle-scene entities, spawning effects, changing positions; required for scene mods |
| `TaleWorlds.Engine.AutoGenerated` | Generated engine bindings | Internal, not used directly |
| `TaleWorlds.Engine.GauntletUI` | Engine-side Gauntlet integration | Internal |
| `TaleWorlds.NavigationSystem` | Navmesh/pathfinding | Custom AI pathfinding, navmesh edits |

### Layer 4 — Gauntlet UI

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.GauntletUI` | Gauntlet UI framework: `Widget`, `ViewModel` two-way binding, `GauntletMovie`, animation | Foundation of any custom UI; new screens, HUDs, menus |
| `TaleWorlds.GauntletUI.Data` | Data binding layer | Pairs with `ViewModel` for data binding |
| `TaleWorlds.GauntletUI.PrefabSystem` | Prefab UI | Loading `.prefab` files, reusing official UI templates |
| `TaleWorlds.GauntletUI.ExtraWidgets` | Extra widgets | When you need controls not provided by the base set |
| `TaleWorlds.GauntletUI.CodeGenerator` | VM code generation | Toolchain, used at build time |

### Layer 5 — SaveSystem

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.SaveSystem` ★ | Save system: `SaveManager`, `SaveContext`, `LoadContext`, `DefinitionContext`, `[SaveableField/Property]` attributes | Any mod that must persist data across sessions (most campaign mods); tag fields and the system serializes them automatically |

### Layer 6 — MountAndBlade

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.MountAndBlade` | Battle core: `Mission`, `Agent`, `Formation`, `Team`, `MissionBehavior`/`MissionLogic`, component system | Core of every battle mod; damage tweaks, AI changes, spawning, mission logic |
| `TaleWorlds.MountAndBlade.AutoGenerated` | Generated bindings | Internal |
| `TaleWorlds.MountAndBlade.ViewModelCollection` | M&B ViewModels | Battle UI data binding |
| `TaleWorlds.MountAndBlade.GauntletUI.Widgets` | M&B Gauntlet widgets | Reusing official battle widgets |
| `TaleWorlds.MountAndBlade.Helpers` | Helpers | Looking up official algorithm implementations (e.g. damage formulas) |

### Layer 7 — Campaign

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.CampaignSystem` | Campaign core: `MobileParty`/`Hero`/`Kingdom`/`Clan`/`Settlement`/`Town`/`Village`/`Campaign`/`CampaignBehavior` | Core of every campaign mod — economy, diplomacy, clans, quests, daily ticks all live here |
| `TaleWorlds.CampaignSystem.ViewModelCollection` | Campaign ViewModels | Campaign map UI, Encyclopedia page data binding |

### Layer 8 — Platform & services

| Module | Responsibility | Mod relevance (when you touch it) |
|--------|---------------|-----------------------------------|
| `TaleWorlds.ModuleManager` | Module loading/dependency resolution `ModuleInfo`/`ModuleHelper` | Writing `SubModule.xml`, declaring dependencies; the mod startup entry |
| `TaleWorlds.Diamond` + `.AccessProvider.*` | Online services/auth (Steam/GOG/GDK) | Rarely used directly |
| `TaleWorlds.MountAndBlade.Diamond` | M&B online/multiplayer services | Multiplayer mods |
| `TaleWorlds.MountAndBlade.Launcher` / `.Library` / `.Steam` | Launcher | Rare; launcher extensions |
| `TaleWorlds.PlatformService` / `.Steam` / `.Epic` / `.GOG` | Platform abstraction | Cross-platform compatibility |
| `TaleWorlds.PlayerServices` / `ServiceDiscovery.Client` | Online services | Rare |
| `TaleWorlds.AchievementSystem` / `ActivitySystem` | Achievements/activities | Custom achievement mods |
| `TaleWorlds.PSAI` | AI | Reference when tuning battle AI |
| `TaleWorlds.Starter.Library` | Startup entry | Rare |
| `TaleWorlds.MountAndBlade.SteamWorkshop` | Workshop | Publishing mods to Steam Workshop |
| `TaleWorlds.MountAndBlade.Multiplayer.Test` / `SaveSystem.CodeGenerator` | Test/tooling | Toolchain |

> v1.3.0 additionally ships **gameplay modules**: `SandBox`(+.GauntletUI/.View/.ViewModelCollection) and `StoryMode`(+...). These are not separately provided in the 1.3.15 source bundle, but are available again in 1.4.5 (see [Version Delta](./version-delta)).

## Version delta quick reference

| Dimension | v1.3.0 | v1.3.15 (this doc) | v1.4.5 |
|-----------|--------|--------------------|--------|
| TaleWorlds module count | 31 | 54 | 65 (DLL) |
| Key change | Includes SandBox/StoryMode gameplay modules | `SaveSystem` split out; platform services split; 23 new modules | Multiplayer/View/Gauntlet split into standalone assemblies; new `CampaignSystem.FastMode`, `BirthAndDeath`, `CustomBattle`, `Platform.PC` |
