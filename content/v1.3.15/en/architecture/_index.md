# Architecture

> Treat architecture as a **map**, not a parts list. This page shows how a mod flows through the SDK and which doc to read next.

## One-sentence map

```
SubModule.xml + DLL  →  ModuleManager  →  MBSubModuleBase  →  Campaign / Mission
        ↓                                      ↓
    XML / assets                          Behavior / ViewModel registration
        ↓                                      ↓
   MBObjectManager  ←  Core  ←  Engine scene  ←  Mission runtime
        ↓                    ↓
   Campaign tick        SaveSystem persistence
```

**90% of mods only touch the top four rows**.

## Layer Cheat Sheet

```
Layer 8  Platform & Services   →  SubModule.xml / ModuleManager / Platform services
Layer 7  Campaign              →  Campaign / Hero / MobileParty / Kingdom / Settlement
Layer 6  MountAndBlade         →  Mission / Agent / Formation / Team / MissionBehavior
Layer 5  SaveSystem            →  SaveManager / SaveContext / LoadContext
Layer 4  Gauntlet UI           →  GauntletMovie / ViewModel / Widget
Layer 3  Engine                →  Scene / GameEntity / NavigationSystem
Layer 2  Core data             →  ItemObject / CharacterObject / MBObjectManager
Layer 1  Systems base          →  ObjectSystem / ScreenSystem / InputSystem
Layer 0  Foundation            →  Library / Localization
```

See [SDK Overview](./sdk-overview) for details.
