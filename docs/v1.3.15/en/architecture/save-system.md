# Save System

**Namespace**: TaleWorlds.SaveSystem
**Version**: 1.3.15 (NEW - Does not exist in 1.3.0)

## Overview

TaleWorlds.SaveSystem is a new modular save system added in version 1.3.15. In 1.3.0, save logic was scattered across modules, but 1.3.15 unified it into a dedicated system with 100+ .cs files.

## Module Structure

```
TaleWorlds.SaveSystem/
├── SaveManager.cs              # Save manager (main entry)
├── GameData.cs                 # Save data structure
├── MetaData.cs                 # Metadata

├── Definition/                 # Type definitions
```

## Core Classes

### SaveManager

```csharp
public static class SaveManager
{
    // Initialize global definition context
    public static void InitializeGlobalDefinitionContext()
    {
        _definitionContext = new DefinitionContext();
        _definitionContext.FillWithCurrentTypes();
    }
    
    // Save game
    public static SaveOutput Save(
        object target,           // Object to save
        MetaData metaData,       // Metadata
        string saveName,         // Save name
        ISaveDriver driver)      // Save driver
    {
        _isLoading = false;
        ISaveContext saveContext = new SaveContext(_definitionContext);
        
        if (saveContext.Save(target, metaData, out string error))
        {
            // Async save
            Task<SaveResultWithMessage> task = driver.Save(
                saveName, 1, metaData, saveContext.SaveData);
            // ...
        }
    }
    
    // Load game
    public static LoadResult Load(string saveName, ISaveDriver driver)
    {
        return Load(saveName, driver, false);
    }
    
    public static LoadResult Load(
        string saveName, 
        ISaveDriver driver, 
        bool loadAsLateInitialize)
    {
        _isLoading = true;
        DefinitionContext definitionContext = new DefinitionContext();
        definitionContext.FillWithCurrentTypes();
        
        LoadData loadData = driver.Load(saveName);
        LoadContext loadContext = new LoadContext(definitionContext, driver);
        
        if (loadContext.Load(loadData, loadAsLateInitialize))
        {
            return LoadResult.CreateSuccessful(
                loadContext.RootObject, 
                loadData.MetaData, 
                loadCallbackInitializator);
        }
    }
}
```

### GameData

The actual save data container:

```csharp
public class GameData
{
    public byte[] Header { get; internal set; }      // Header info
    public byte[] Strings { get; internal set; }    // String data
    public byte[][] ObjectData { get; internal set; }   // Object data
    public byte[][] ContainerData { get; internal set; } // Container data
    
    public int TotalSize => Header.Length + Strings.Length 
        + ObjectData.Sum(d => d.Length) 
        + ContainerData.Sum(d => d.Length);
}
```

## Save Flow

```
Save triggered (User clicks "Save" in menu)
    │
    ▼
Module.OnGameEnd()
    │
    ▼
Game.OnFinalize()
    │
    ▼
SaveManager.Save(target, metaData, saveName, driver)
    │
    ├─► Create SaveContext
    │
    ├─► DefinitionContext checks type definitions
    │
    ├─► SaveContext.Save() serializes objects
    │       │
    │       ├─► Iterate all [SaveableField]/[SaveableProperty]
    │       │
    │       ├─► Call corresponding Serializer
    │       │
    │       └─► Generate GameData (Header + Strings + ObjectData + ContainerData)
    │
    └─► ISaveDriver.Save() writes file
            │
            └─► AsyncFileSaveDriver → ZIP compress → Write .sav file
```

## Load Flow

```
Load triggered (User clicks "Continue" or selects a save)
    │
    ▼
SaveManager.Load(saveName, driver)
    │
    ├─► Create new DefinitionContext
    │
    ├─► ISaveDriver.Load() reads .sav file
    │       │
    │       └─► Decompress → GameData
    │
    ├─► Create LoadContext
    │
    ├─► LoadContext.Load() deserializes
    │       │
    │       ├─► Read Header to reconstruct object structure
    │       │
    │       ├─► Read ObjectData/ContainerData
    │       │
    │       └─► Parse string table
    │
    └─► Return LoadResult
            │
            ├─► RootObject (reconstructed game object)
            ├─► MetaData (save metadata)
            └─► LoadCallbackInitializator (late initialize callback)
```

## Save File Format

Save files (.sav) are ZIP archives containing:

```
save.zip
├── header.bin      # Binary header
├── strings.bin     # String table
├── objects_0.bin   # Object data blocks
├── objects_1.bin   # More object data
├── container_0.bin # Container data blocks
└── container_1.bin # More container data
```

## Usage Example

```csharp
// Define saveable class
[SaveableRootClass(5000)]
public class MyGameData
{
    [SaveableField(1)]
    public int Score;
    
    [SaveableField(2)]
    public string PlayerName;
    
    [SaveableProperty(3)]
    public List<Item> Inventory { get; set; }
}

// Save game
public void SaveGame(string saveName)
{
    MyGameData data = new MyGameData { Score = 100, PlayerName = "Hero" };
    MetaData meta = new MetaData 
    { 
        SaveName = saveName,
        GameVersion = ApplicationVersion.FromParametersFile(null)
    };
    
    ISaveDriver driver = new AsyncFileSaveDriver();
    SaveOutput result = SaveManager.Save(data, meta, saveName, driver);
    
    if (result.Succeeded)
    {
        Debug.Print("Game saved successfully!");
    }
}

// Load game
public void LoadGame(string saveName)
{
    ISaveDriver driver = new AsyncFileSaveDriver();
    LoadResult result = SaveManager.Load(saveName, driver);
    
    if (result.Succeeded)
    {
        MyGameData data = result.RootObject as MyGameData;
        Debug.Print($"Loaded: {data.PlayerName}, Score: {data.Score}");
    }
}
```

## 1.3.0 vs 1.3.15 Comparison

| Feature | 1.3.0 | 1.3.15 |
|---------|-------|--------|
| Save System | Scattered in modules | TaleWorlds.SaveSystem |
| Type Definition | Runtime reflection | DefinitionContext pre-registration |
| Serialization | Manual implementation | Attribute-based automatic |
| Async Save | Not supported | AsyncFileSaveDriver |
| Save Format | Binary | ZIP (Header+Strings+Object+Container) |
| Code Generation | None | AutoGeneratedSaveManager (1184 lines) |
