# 架构总览

> **把架构当作一张交通图，而不是一张零件清单。**  本文告诉你：一个 mod 从启动到存档会经过哪些系统、为什么要经过它们、以及下一步该读哪一页。

## 一句话地图

```
SubModule.xml + DLL  →  ModuleManager  →  MBSubModuleBase  →  Campaign/Mission
        ↓                                      ↓
    XML/资源加载                          Behavior/ViewModel 注册
        ↓                                      ↓
   MBObjectManager  ←  Core 数据  ←  Engine 场景  ←  Mission 运行
        ↓                    ↓
   Campaign 每日 Tick   SaveSystem 持久化
```

**90% 的 mod 只和最上四行打交道**：Module 入口 → Behavior → 战役/任务逻辑 → 存档。

## 层次速记 / Layer Cheat Sheet

```
Layer 8  平台与服务  →  SubModule.xml / ModuleManager / Steam/Epic 服务
Layer 7  战役         →  Campaign / MobileParty / Kingdom / Clan / Settlement
Layer 6  战斗         →  Mission / Agent / Formation / Team / MissionBehavior
Layer 5  存档         →  SaveManager / SaveContext / LoadContext
Layer 4  UI          →  Gauntlet / ViewModel / GauntletMovie
Layer 3  引擎         →  Scene / GameEntity / NavigationSystem
Layer 2  核心数据      →  ItemObject / CharacterObject / MBObjectManager
Layer 1  系统基座      →  InputSystem / ScreenSystem / ObjectSystem
Layer 0  基础库        →  Library / Localization
```

详见 [SDK 总览](./sdk-overview)。
