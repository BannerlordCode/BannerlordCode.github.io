## Search Guidance

1. If you know a managed interface method, start with the `[EngineMethod]` entry in the [Native Reference](../native/), then search for the native method name or nearby registration code.
2. If you know a native type name, search strings such as `ftdnNative_string`, `rglScene`, or `rglMaterial`, then follow nearby vftable writes and constructors.
3. If you know a function address, confirm it in [ALL-FUNCTIONS-LIST.txt](./ALL-FUNCTIONS-LIST.txt), then search the generated `FUN_` name in the decompiled source.
4. If you have a crash log or diagnostic text, search the literal string first. Examples include `Unable to find native module`, `/nativeTest`, and `Modules/Native/get_tileset.bat`.
5. For version comparisons, do not treat addresses as stable API. Addresses are only valid for the current v1.3.15 decompiled sample.
