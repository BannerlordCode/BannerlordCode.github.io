# Native Reference

Complete documentation for interacting with the native C++ engine via P/Invoke.

## Architecture Overview

```
C++ (TaleWorlds.Native.dll)
    ↓ DllImport: TaleWorlds.Starter.Library/MBDotNet.cs
    ↓ PassManagedEngineCallbackMethodPointersDotNet()
    ↓ Function pointers (IntPtr)
ScriptingInterfaceObjects.SetFunctionPointer(int id, IntPtr ptr)
    ↓ Static delegates
INativeArray / INativeString / IManaged
    ↓ Managed wrappers
NativeArray<T>, NativeString, NativeObject
    ↓ User code
Your mod code
```

## Core Concepts

### 1. P/Invoke Call Flow

1. **Native side** (C++) exports functions, passes function pointers via `pass_managed_library_callback_method_pointers`
2. **ScriptingInterfaceObjects** receives function pointers, converts to static delegates
3. **INative*** interface methods marked with `[EngineMethod]` attribute
4. **Native*** wrapper classes call interface methods
5. **User code** uses Native* wrapper classes

### 2. Memory Management

| Role | Responsibility |
|------|----------------|
| **Native side** | Allocates/frees native object memory |
| **IManaged** | Reference counting management |
| **NativeObject** | Finalizer auto-calls `DecreaseReferenceCount` |
| **ManualInvalidate()** | Active release (optional) |

### 3. Key Attributes

- `[EngineClass("engine_type")]` - Mark class corresponding to native type
- `[EngineMethod("native_method")]` - Mark method corresponding to native function
- `[LibraryInterfaceBase]` - Mark interface as native interface

## Usage Example

```csharp
// Create native array
NativeArray array = NativeArray.Create();
array.AddElement(42);
array.AddElement(3.14f);
int[] managedArray = array.ToArray<int>();
array.Clear();

// Operate native string
NativeString str = NativeString.Create();
str.SetString("Hello, Bannerlord!");
string result = str.GetString();

// Native object array
NativeObjectArray objArray = NativeObjectArray.Create();
objArray.AddElement(someNativeObject);
int count = objArray.Count;
```

## Important Notes

1. **Do not manually modify AutoGenerated files** - `TaleWorlds.DotNet.AutoGenerated/` and `TaleWorlds.MountAndBlade.AutoGenerated/` are tool-generated
2. **Pointer type** - Use `UIntPtr` not `IntPtr` for storing native pointers
3. **Reference counting** - Each NativeObject increments ref count on construction, decrements on destruction
4. **Thread safety** - Native interfaces accessed via static fields, be careful of threading
