# SDK 总览 — 模块地图与开发者路线图

## 心智模型

先把 `SDK 总览 — 模块地图与开发者路线图` 当作这个子系统的入口或数据节点来理解：先看属性代表什么状态，再看方法允许你做什么。

> 本文是整份文档的**导航中枢**。它不只是一张模块清单——而是从开发者视角回答三个问题：每一层是**为什么存在**、你**什么时候会碰到**它、以及如何**从这里跳到**对应的 API 文档。读完本文你应当能在一分钟内判断「我的需求要碰哪几层、入口类是谁、文档在哪」。

## 一句话定位

Bannerlord 的托管代码是一组**严格分层**的 `TaleWorlds.*` 程序集：底层基础库 → 引擎桥接 → 核心数据 → UI 框架 → 存档 → 游戏逻辑（Mission + Campaign）→ 平台/服务。**mod 开发者 90% 的工作集中在最上层的 Campaign 与 Mission，以及 UI 与存档四个层面；下四层只在「读输入、做数学、查场景」等基础场景才需要直接接触。**

## 分层依赖图

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 8  平台与服务 Platform & Services                       │
│   ModuleManager · Diamond · PlatformService(.Steam/.Epic/.GOG)│
│   PlayerServices · ServiceDiscovery · Launcher · Diamond(M&B) │
├─────────────────────────────────────────────────────────────┤
│ Layer 7  战役 Campaign  ← mod 最高频                            │
│   CampaignSystem · CampaignSystem.ViewModelCollection         │
├─────────────────────────────────────────────────────────────┤
│ Layer 6  战斗 MountAndBlade  ← mod 最高频                       │
│   MountAndBlade · .AutoGenerated · .ViewModelCollection       │
│   .Helpers · .GauntletUI.Widgets                              │
├─────────────────────────────────────────────────────────────┤
│ Layer 5  存档 SaveSystem  ← mod 持久化                          │
│   SaveSystem (SaveManager / SaveContext / LoadContext)        │
├─────────────────────────────────────────────────────────────┤
│ Layer 4  UI 框架 Gauntlet                                      │
│   GauntletUI · .Data · .PrefabSystem · .ExtraWidgets · .CodeGen│
├─────────────────────────────────────────────────────────────┤
│ Layer 3  引擎桥接 Engine                                       │
│   Engine · .AutoGenerated · .GauntletUI · NavigationSystem    │
├─────────────────────────────────────────────────────────────┤
│ Layer 2  核心数据 Core                                         │
│   Core (ItemObject/CharacterObject/Equipment) · Core.VMCollection│
├─────────────────────────────────────────────────────────────┤
│ Layer 1  系统基座 Systems                                      │
│   ObjectSystem · ScreenSystem · InputSystem · DotNet ·        │
│   TwoDimension · Network                                      │
├─────────────────────────────────────────────────────────────┤
│ Layer 0  基础库 Foundation                                     │
│   Library · LinQuick · Localization                           │
└─────────────────────────────────────────────────────────────┘
        ↕ 全程依赖 Native 引擎 (TaleWorlds.Native.dll)
```

依赖方向**自下而上**：上层引用下层，下层绝不反向引用上层。mod 代码位于最顶层，可触及任意层。**新手只看上四层即可起步**；下四层是支撑基础设施，遇到具体 API 时再回头查。

## 开发者路线图

> 这是本文档的中心。**「我想做 X，该碰哪几层？」** 下表把常见 mod 目标映射到层、入口类和文档入口。表中链接全部是相对路径，方便你在不同版本/语言间切换。

| 我想做的事 | 碰这几层 | 入口类 / 机制 | 文档起点 |
|-----------|---------|--------------|---------|
| 注册一个 mod（游戏启动钩子） | L8 ModuleManager + L6 M&B | `MBSubModuleBase`、`ModuleInfo` | [模块系统](./module-system) |
| 加新兵种 / 物品 / 装备 | L2 Core + XML | `ItemObject`、`CharacterObject`、`Equipment` + `MBObjectManager` | [Core API](../api/core/) · [Items API](../api/items/) |
| 做战役 mod（每日 tick、行为、王国逻辑） | L7 Campaign | `CampaignBehaviorBase` / `CampaignBehavior`、`CampaignEvents` | [Campaign API](../api/campaign/) · [战役指南](../guide/campaign-system) |
| 做战斗 / 任务 mod（AI、伤害、生成） | L6 MountAndBlade | `MissionBehavior`、`MissionLogic`、`Agent`、`Formation` | [Mission API](../api/mission/) · [任务指南](../guide/mission-system) |
| 加自定义 UI 屏（HUD、菜单、对话框） | L4 Gauntlet + L6 M&B | `ViewModel` + `GauntletMovie`、`ScreenBase` | [GUI API](../api/gui/) · [Gauntlet 指南](../guide/gauntlet-ui) |
| 持久化自定义数据到存档 | L5 SaveSystem | `[SaveableField]` / `[SaveableProperty]` + `SaveManager` | [Save API](../api/save-system/) · [存档架构](./save-system) |
| 加本地化文本（多语言字符串） | L0 Localization | `TextObject`、`MBTextManager` | [Localization API](../api/localization/) · [本地化指南](../guide/localization) |
| 操作场景实体（位置、骨骼、特效） | L3 Engine | `Scene`、`GameEntity`、`MatrixFrame`、`Skeleton` | [Engine API](../api/engine/) |
| 读取玩家输入（键鼠 / 手柄） | L1 InputSystem | `InputContext`、`InputKey` | [API 总览](../api/) |
| 加新的 ViewModel / 数据绑定 | L4 Gauntlet + L1 Core.VMCollection | `ViewModel`、`DataSourceProperty` | [ViewModel API](../api/viewmodel/) |
| 自定义战役扩展类型（领地、模板） | L7 Campaign + L2 Core | `Settlement`、`PartyTemplateObject`、`CultureObject` | [Campaign-Ext API](../api/campaign-ext/) |
| 自定义战斗组件 / Agent 行为 | L6 MountAndBlade 扩展 | `AgentComponent`、`MissionLogic` 子类 | [Mission-Ext API](../api/mission-ext/) |
| 扩展核心模型（锻造、技能、概念） | L2 Core 扩展 | `SkillObject`、`CraftingTemplate`、`BasicCharacterObject` | [Core-Extra API](../api/core-extra/) |
| 多人模式底层（少见，1.4.5 拆分后） | L8 Network + L6 M&B.Multiplayer | `MultiplayerMission` 系列 | [版本差异](./version-delta) |

> **使用方式**：先确定你的目标行 → 跳到对应 API 文档读入口类签名 → 再回这里查依赖层是否需要配合（例如战役 mod 几乎都需要 L5 存档）。

## 模块清单（v1.3.15，按层分组）

> 规模：**54 个 TaleWorlds 模块 / 5196 个 .cs 文件 / 5811 个类型**。★ 标记为 1.3.15 新增。「mod 相关性」列给出**具体使用场景**而非抽象评分，方便你判断是否需要深入。

### Layer 0 — 基础库 Foundation

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.Library` | 通用工具：`Vec3`/`Mat3`/`Vec2` 数学、`MBList`/`MBReadOnlyList`、线程、IO、回调、`InformationManager` 消息、调试 | 天天用——几乎每个文件都 `using TaleWorlds.Library`；显示信息提示框、做向量数学、写调试日志 |
| `TaleWorlds.LinQuick` | LINQ 性能优化替代实现 | 内部用，mod 一般不直接引用 |
| `TaleWorlds.Localization` | 本地化：`TextObject`、`MBTextManager`、语言数据 | 所有面向玩家显示的文本必经；自定义字符串、多语言 mod |

### Layer 1 — 系统基座 Systems

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.ObjectSystem` | `MBObjectManager`、`MBObjectBase`、XML 反序列化与对象注册表 | 从 XML 加载自定义物品/角色时；`MBObjectManager.RegisterObject` |
| `TaleWorlds.ScreenSystem` | 屏幕栈管理 `ScreenBase`/`ScreenManager` | 自定义全屏 UI、覆盖默认屏幕时 |
| `TaleWorlds.InputSystem` | 输入抽象 `InputContext`、按键/手柄 | 读取玩家按键、做自定义快捷键 mod |
| `TaleWorlds.DotNet` / `.AutoGenerated` | 托管/原生互操作原语、只读字典等 | 内部，极少直接用 |
| `TaleWorlds.TwoDimension` / `.Standalone` | 2D 渲染：精灵、字体、画布 | 引擎内部，做底层 2D 渲染才碰 |
| `TaleWorlds.Network` | 网络原语 | 多人 mod 底层；1.4.5 多人拆分后更常用 |

### Layer 2 — 核心数据 Core

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.Core` | 核心域模型：`ItemObject`/`Equipment`/`CharacterObject`/锻造/基础概念/信息通道 | 定义新物品、新兵种、新装备；战役与战斗都会触及 |
| `TaleWorlds.Core.ViewModelCollection` | 核心 ViewModel | 做核心数据绑定时配合 L4 使用 |

### Layer 3 — 引擎桥接 Engine

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.Engine` | 引擎桥接：`Scene`、`GameEntity`、`NativeObject`、`MatrixFrame`、`Skeleton`、`Utilities` | 操作战斗场景实体、放特效、改位置；场景类 mod 必经 |
| `TaleWorlds.Engine.AutoGenerated` | 自动生成的引擎绑定 | 内部，不直接用 |
| `TaleWorlds.Engine.GauntletUI` | 引擎侧 Gauntlet 接入 | 内部 |
| `TaleWorlds.NavigationSystem` | 导航网格/寻路 | 做自定义 AI 寻路、改 navmesh 时 |

### Layer 4 — UI 框架 Gauntlet

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.GauntletUI` | Gauntlet UI 框架：`Widget`、`ViewModel` 双向绑定、`GauntletMovie`、动画 | 任何自定义 UI 的基础；新建屏幕、HUD、菜单 |
| `TaleWorlds.GauntletUI.Data` | 数据绑定层 | 配合 `ViewModel` 做数据绑定 |
| `TaleWorlds.GauntletUI.PrefabSystem` | Prefab 预制 UI | 加载 `.prefab` 文件、复用官方 UI 模板 |
| `TaleWorlds.GauntletUI.ExtraWidgets` | 扩展控件 | 需要官方未提供的特殊控件时 |
| `TaleWorlds.GauntletUI.CodeGenerator` | VM 代码生成 | 工具链，构建时用 |

### Layer 5 — 存档 SaveSystem

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.SaveSystem` ★ | 存档系统：`SaveManager`、`SaveContext`、`LoadContext`、`DefinitionContext`、`[SaveableField/Property]` 特性 | 任何需要跨会话保留数据的 mod（绝大多数战役 mod）；标记字段后由系统自动序列化 |

### Layer 6 — 战斗 MountAndBlade

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.MountAndBlade` | 战斗核心：`Mission`、`Agent`、`Formation`、`Team`、`MissionBehavior`/`MissionLogic`、组件系统 | 所有战斗 mod 的核心；改伤害、改 AI、改生成、加任务逻辑 |
| `TaleWorlds.MountAndBlade.AutoGenerated` | 自动生成绑定 | 内部 |
| `TaleWorlds.MountAndBlade.ViewModelCollection` | M&B ViewModel | 战斗 UI 数据绑定 |
| `TaleWorlds.MountAndBlade.GauntletUI.Widgets` | M&B Gauntlet 控件 | 复用官方战斗控件 |
| `TaleWorlds.MountAndBlade.Helpers` | 辅助方法 | 查官方算法实现（如伤害公式） |

### Layer 7 — 战役 Campaign

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.CampaignSystem` | 战役核心：`MobileParty`/`Hero`/`Kingdom`/`Clan`/`Settlement`/`Town`/`Village`/`Campaign`/`CampaignBehavior` | 战役 mod 的核心——经济、外交、家族、任务、每日 tick 全在这里 |
| `TaleWorlds.CampaignSystem.ViewModelCollection` | 战役 ViewModel | 战役地图 UI、 Encyclopedia 页面数据绑定 |

### Layer 8 — 平台与服务

| 模块 | 职责 | mod 相关性（何时会用到） |
|------|------|------------------------|
| `TaleWorlds.ModuleManager` | 模块加载/依赖解析 `ModuleInfo`/`ModuleHelper` | 写 `SubModule.xml`、声明依赖时；mod 启动入口 |
| `TaleWorlds.Diamond` + `.AccessProvider.*` | 在线服务/鉴权（Steam/GOG/GDK） | 极少直接用 |
| `TaleWorlds.MountAndBlade.Diamond` | M&B 在线/多人服务 | 多人 mod |
| `TaleWorlds.MountAndBlade.Launcher` / `.Library` / `.Steam` | 启动器 | 极少；启动器扩展 |
| `TaleWorlds.PlatformService` / `.Steam` / `.Epic` / `.GOG` | 平台抽象 | 跨平台兼容时 |
| `TaleWorlds.PlayerServices` / `ServiceDiscovery.Client` | 在线服务 | 极少 |
| `TaleWorlds.AchievementSystem` / `ActivitySystem` | 成就/活动 | 自定义成就 mod |
| `TaleWorlds.PSAI` | AI | 战斗 AI 调优时参考 |
| `TaleWorlds.Starter.Library` | 启动入口 | 极少 |
| `TaleWorlds.MountAndBlade.SteamWorkshop` | 创意工坊 | 发布 mod 到创意工坊 |
| `TaleWorlds.MountAndBlade.Multiplayer.Test` / `SaveSystem.CodeGenerator` | 测试/工具 | 工具链 |

> v1.3.0 额外包含**玩法模块**：`SandBox`(+.GauntletUI/.View/.ViewModelCollection) 与 `StoryMode`(+...)。这些在 1.3.15 源码包中未单独提供，1.4.5 已重新可用（见 [版本差异](./version-delta)）。

## 版本差异速查

| 维度 | v1.3.0 | v1.3.15（本文档） | v1.4.5 |
|------|--------|------------------|--------|
| TaleWorlds 模块数 | 31 | 54 | 65 (DLL) |
| 关键变化 | 含 SandBox/StoryMode 玩法模块 | `SaveSystem` 独立；平台服务拆分；23 个新模块 | 多人/View/Gauntlet 拆分为独立程序集；新增 `CampaignSystem.FastMode`、`BirthAndDeath`、`CustomBattle`、`Platform.PC` |
| 迁移注意 | — | `SaveSystem` 引用路径变化 | 移除 `Launcher`、`SteamWorkshop`、`CodeGenerator`；多人 mod 需改引用 |

> 模块级完整对照见 [版本差异](./version-delta)；**逐类 API 跨版本对比**（哪些方法被移除/新增）见 [🔀 跨版本类对比](/versions/)。
