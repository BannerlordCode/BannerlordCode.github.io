## 架构概述 / Architecture Overview

```
C++ (TaleWorlds.Native.dll)
    ↓ DllImport: TaleWorlds.Starter.Library/MBDotNet.cs
    ↓ PassManagedEngineCallbackMethodPointersDotNet()
    ↓ Function pointers (IntPtr)
ScriptingInterfaceObjects.SetFunctionPointer(int id, IntPtr ptr)
    ↓ Static delegates
INativeArray / INativeString / IManaged
    ↓ Managed wrappers
NativeArray<T>, NativeString, NativeObject
    ↓ User code
Your mod code
```

## 核心概念 / Core Concepts

### 1. P/Invoke 调用流程 / P/Invoke Call Flow

1. **原生端** (C++) 导出函数，通过 `pass_managed_library_callback_method_pointers` 传递函数指针
2. **ScriptingInterfaceObjects** 接收函数指针，转换为静态委托
3. **INative*** 接口方法使用 `[EngineMethod]` 特性标记
4. **Native*** 包装类调用接口方法
5. **用户代码** 使用 Native* 包装类

### 2. 内存管理 / Memory Management

| 角色 | 职责 |
|------|------|
| **原生端** | 分配/释放原生对象内存 |
| **IManaged** | 引用计数管理 |
| **NativeObject** | Finalizer 自动调用 `DecreaseReferenceCount` |
| **ManualInvalidate()** | 主动释放引用（可选） |

### 3. 关键特性 / Key Attributes

- `[EngineClass("engine_type")]` - 标记类对应原生类型
- `[EngineMethod("native_method")]` - 标记方法对应原生函数
- `[LibraryInterfaceBase]` - 标记接口为原生接口

## 使用示例 / Usage Example

```csharp
// 创建原生数组
NativeArray array = NativeArray.Create();
array.AddElement(42);
array.AddElement(3.14f);
int[] managedArray = array.ToArray<int>();
array.Clear();

// 操作原生字符串
NativeString str = NativeString.Create();
str.SetString("Hello, Bannerlord!");
string result = str.GetString();

// 原生对象数组
NativeObjectArray objArray = NativeObjectArray.Create();
objArray.AddElement(someNativeObject);
int count = objArray.Count;
```

## 注意事项 / Important Notes

1. **不要手动修改 AutoGenerated 文件** - `TaleWorlds.DotNet.AutoGenerated/` 和 `TaleWorlds.MountAndBlade.AutoGenerated/` 由工具生成
2. **指针类型** - 使用 `UIntPtr` 而非 `IntPtr` 存储原生指针
3. **引用计数** - 每个 NativeObject 构造时增加引用，销毁时减少
4. **线程安全** - 原生接口通过 static 字段访问，需要注意线程安全
