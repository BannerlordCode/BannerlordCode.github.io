## 三个核心问题

### Q1：什么对象能存？

任何带有 `[SaveableClass]` 或 `[SaveableRootClass]` 的类都可以进入图。引用这些对象的其他对象如果也被标记，会自动跟随序列化。未被标记的引用会被跳过或报错。

### Q2：用什么标记存？

| 属性 | 用途 | 注意 |
|------|------|------|
| `[SaveableRootClass(id)]` | 根对象，一次保存只能有一个根 | 如 `Campaign` 或自定义全局状态 |
| `[SaveableClass(id)]` | 普通可存档类 | id 在同一个定义上下文中必须唯一 |
| `[SaveableField(id)]` | 可存档字段 | id 在类内唯一 |
| `[SaveableProperty(id)]` | 可存档属性 | 需要公开 getter/setter 或指定访问级别 |

示例：

```csharp
[SaveableClass(12345)]
public class MyModState
{
    [SaveableField(1)]
    public int Reputation;

    [SaveableProperty(2)]
    public List<Hero> FavoredHeroes { get; set; }
}
```

### Q3：存到哪里去？

由 `ISaveDriver` 决定：

```csharp
public interface ISaveDriver
{
    Task<SaveResultWithMessage> Save(string saveName, int version, MetaData metaData, GameData gameData);
    LoadData Load(string saveName);
}
```

游戏内默认实现是 `AsyncFileSaveDriver`：把 `GameData` 压缩成 `.sav` ZIP。

## 保存与加载流程

```
保存
用户点击“保存”
   │
   ▼
SaveManager.Save(rootObject, metaData, saveName, driver)
   │
   ├── SaveContext 遍历 [SaveableX] 字段
   ├── DefinitionContext 验证类型/字段 ID
   ├── 生成 GameData (Header + Strings + ObjectData + ContainerData)
   └── driver.Save(...) → ZIP → .sav

加载
用户选择存档
   │
   ▼
SaveManager.Load(saveName, driver)
   │
   ├── driver.Load(...) → ZIP → GameData
   ├── 新建 DefinitionContext（基于当前程序集类型）
   ├── LoadContext 反序列化
   └── 返回 LoadResult { RootObject, MetaData }
```

迁移场景：**v1.3.0 → v1.3.15** 时 `SaveSystem` 命名空间/类型名没有变，但 DLL 引用需要改为独立的 `TaleWorlds.SaveSystem.dll`。

## DefinitionContext 与 AutoGeneratedSaveManager

- `DefinitionContext` 维护“类型 ID → 类型定义”的映射。它在启动时 `FillWithCurrentTypes()` 收集当前 AppDomain 中所有 `[Saveable*]` 类型。
- `AutoGeneratedSaveManager` 在 `FillWithCurrentTypes` 之后注入容器与数组的序列化委托，如 `List<T>`、`Dictionary<TKey,TValue>`、`T[]`。这也是为什么自定义泛型容器通常能自动保存的原因。

不要手动调用 `AutoGeneratedSaveManager`，但要理解：**如果自定义类型包含复杂容器且读档报错，先检查容器中元素是否也是 `[SaveableClass]`**。

## 存档文件格式

`.sav` 是 ZIP，内部包含：

```
save.zip
├── header.bin         # 对象结构与类型 ID 表
├── strings.bin        # 字符串表
├── object_0.bin       # 对象实例数据
├── container_0.bin    # List/Dict/Array 数据
└── ...
```

## 实战：把自定义数据存进战役存档

```csharp
public class MyCampaignBehavior : CampaignBehaviorBase
{
    private MyModState _state;

    public override void RegisterEvents()
    {
        CampaignEvents.OnSessionLaunchedEvent.AddNonSerializedListener(this, OnSessionLaunched);
        CampaignEvents.DailyTickEvent.AddNonSerializedListener(this, DailyTick);
    }

    private void OnSessionLaunched(CampaignGameStarter starter)
    {
        _state = new MyModState { Reputation = 0 };
    }

    private void DailyTick()
    {
        _state.Reputation++;
    }

    public override void SyncData(IDataStore dataStore)
    {
        dataStore.SyncData("MyModState", ref _state);
    }
}
```

同时给 `MyModState` 标记 `[SaveableClass]` 与 `[SaveableField/Property]`。

## 关键类速查

| 类 | 职责 |
|----|------|
| `SaveManager` | 保存/加载入口 |
| `SaveContext` / `LoadContext` | 序列化/反序列化上下文 |
